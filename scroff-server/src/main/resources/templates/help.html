📖 Scroff 使用教程

5 分钟上手集中控屏系统

1. Scroff 是什么？

Scroff（Screen On/Off）是一个通过 ADB over TCP/IP 远程管理安卓叫号机屏幕开关的服务。 你只需要把叫号机的 IP 配进来，就能在 Web 后台一键开关、定时开关，不用再跑到现场。

📺 适用场景：医院、银行、政府大厅等场所的多台安卓叫号机/信息屏， 需要统一开关、定时开关。

2. 快速开始（5 分钟）

第 1 步：把叫号机切到 TCP/IP 模式

每台叫号机只需做一次（之后再也不需要 USB）：

  1. 用 USB 把叫号机连到一台电脑（Mac/Win/Linux 都行）
  2. 执行：adb tcpip 5555 （设备重启后会保持 TCP 模式）
  3. 拔掉 USB
  4. 在同网段任何机器执行：adb connect 192.168.1.50:5555
💡 提示：叫号机只要和 Scroff Server 在同一子网就行。 如果叫号机是 Android 11+，无需 USB 也能开启 ADB 调试（开发者选项 → 无线调试）。

第 2 步：在 Scroff 添加设备

  1. 登录 Scroff 后台 → 设备 → + 新建设备
  2. 填写：
    • 名称：方便识别的名字（如"前台叫号机 1"）
    • 分类：分组管理（如"前台"/"取药"/"检验"）
    • IP：叫号机的局域网 IP（如 192.168.1.50）
    • 端口：默认 5555
    • 排序：列表里显示的顺序，数字小的在前
  3. 点 保存

第 3 步：测试连接

回到设备列表，点 测连 按钮：

  • 🟢 绿色 ONLINE：连接成功，可以控制
  • 🔴 红色 OFFLINE：连不上，鼠标悬停看错误详情
⚠️ 常见连接失败：
  • 叫号机没在同一子网（路由隔离）
  • 叫号机防火墙阻断了 5555 端口
  • 叫号机重启后没保留 TCP 模式（部分定制 ROM 会重置）

第 4 步：立即控制

在设备列表点 操作 ▼ 下拉菜单：

  • 开屏 / 关屏：立即发送 ADB 命令
  • 查屏状态：弹出 modal，显示当前屏幕实际开关状态
  • 测试所有连接 / 查所有屏状态：批量操作（顶部按钮）

3. 定时任务（自动化）

两种模式

模式 说明 典型场景
单台设备 只对一台叫号机生效 某台机需要特殊作息（如自助机 24h 不关）
所有设备 对所有启用的设备生效 每天统一 7:50 开屏、17:50 关屏

⏰ 优先级规则：单台 > 所有

当一台叫号机同时被两个 schedule "管到"时：

  • 所有设备 schedule：0 50 17 * * *（每天 17:50 关屏）
  • 叫号机 1 的单台 schedule：0 50 17 * * *（每天 17:50 关屏）

17:50 这一刻，叫号机 1 只由单台 schedule 控制（不走所有设备那条），其他设备走所有设备那条。

匹配条件：必须 cron 字符串完全相同（同秒同分同时刻）才视为"冲突"。 如果单台是 17:55、所有是 17:50，两者不冲突，叫号机 1 会被所有 schedule 关一次，再被单台 schedule 单独关一次。

Cron 表达式（6 段）

格式：秒 分 时 日 月 周

表达式 含义
0 50 7 * * *每天 07:50
0 50 17 * * *每天 17:50
0 0 8 * * 1-5工作日（周一到周五）08:00
0 0 22 * * *每天 22:00
0 0 0 1 * *每月 1 号 00:00

表单里有"常用预设"按钮，点一下自动填入，不用手写。

4. 高级功能

多 ADB 路径（按环境切换）

Scroff 支持配置多个 ADB profile（如 Windows 上的 adb.exe 和 Linux 上的 /usr/bin/adb，或通过 WSL 调用）。 在 设置 页面可一键切换，立即生效，无需重启。

分类与排序

  • 每个设备有"分类"字段（自由文本，如"前台"/"取药"）
  • "排序"字段数字小的在前，方便按业务顺序展示
  • 复制设备时 sortOrder 自动 +1，新设备排在源设备后面

页面大小

设备列表右上角有"每页 N 条"下拉，10/20/50/100/200 可选。 选择会写入浏览器 localStorage，刷新页面保留。

5. 故障排查

"OFFLINE" / 测连失败

  1. 叫号机和 Server 在同一子网？ping 192.168.1.50
  2. 叫号机 5555 端口可达？telnet 192.168.1.50 5555
  3. 叫号机的 ADB 调试开着？开发者选项 → USB 调试
  4. Server 端 adb 路径正确？设置 页面检查

定时任务不触发

  1. 看 日志：有没有 schedule 触发: id=X 的记录？
  2. Server 进程是否在跑？容器/CI 环境可能不持久
  3. Server 时区对吗？date 看服务器时间

关屏命令不生效

Scroff 默认用 dumpsys power 控屏，部分定制 ROM（如某些医疗设备） 可能屏蔽了此命令。在 application.yml 里改 scroff.adb.screen-off-command 即可适配。

6. 安全注意

  • ⚠️ Scroff 后台无内置登录，生产环境必须放在内网，或加 Nginx 反向代理 + Basic Auth
  • 数据库密码、ADB 路径等敏感配置写在 application-local.yml，不会提交到 Git
  • 删除设备前，先到 定时 页面删掉引用该设备的任务
