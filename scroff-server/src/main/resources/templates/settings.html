ADB 配置

不同环境需要不同的 adb 可执行路径。配置多个 profile 后，可在此页面即时切换激活项， 选择会写入 system_config 表，重启不丢。

状态 id 名称 可执行路径 args yaml enabled 路径可执行? 操作
未在 application.yml 配置任何 ADB profile。请先在 yaml 里添加。
ACTIVE - id name /path/to/adb arg - 是 ✓
使用中
当前激活： id — /path/to/adb + args: arg
点此按钮后，下次启动会重新读 yaml 的 activeProfileId

配置示例

在 application.yml 或 application-local.yml 的 scroff.adb 下添加：

scroff:
  adb:
    profiles:
      - id: android-sdk
        name: "Android SDK (Windows)"
        executable: "C:/Users/your-name/AppData/Local/Android/Sdk/platform-tools/adb.exe"
        enabled: true
      - id: wsl-ubuntu
        name: "WSL Ubuntu adb"
        executable: "C:/Windows/System32/wsl.exe"
        args: ["adb"]
        enabled: false
      - id: linux-server
        name: "Linux Server"
        executable: "/usr/bin/adb"
        enabled: false
    active-profile-id: android-sdk   # 可选：默认激活哪个
    auto-connect-on-startup: true
    heartbeat-interval: 30
    command-timeout: 10000
